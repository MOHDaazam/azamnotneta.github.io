MySQL Interview Questions & Answers by Mohd Azam

Entry, Mid and Senior Level

  • Explain the difference between INNER JOIN and LEFT JOIN.

    • INNER JOIN returns rows where there is a match in both tables, while LEFT JOIN returns all rows from the left table and matching rows from the right table (null for non-matching rows).
    • Example Query: 
      SELECT employees.name, departments.department_name
        FROM employees
        INNER JOIN departments ON employees.department_id = departments.department_id;

  • Discuss the ACID properties in the context of database transactions.

    • ACID stands for Atomicity, Consistency, Isolation, and Durability. These properties ensure that database transactions are reliable and maintain data integrity.
    • Example Query: N/A

  • Explain the purpose of the GROUP BY clause in SQL.

    • GROUP BY is used to group rows based on specified columns, often used with aggregate functions like COUNT, SUM, AVG, etc.
    • Example Query: 
      SELECT department, COUNT(*) as employee_count
        FROM employees
        GROUP BY department;

  • What is the difference between UNION and UNION ALL?

    • UNION combines and removes duplicate rows, while UNION ALL combines all rows, including duplicates.
    • Example Query: 
      SELECT column1 FROM table1
        UNION
        SELECT column1 FROM table2;

  • How can you optimize SQL queries for better performance?

    • Optimization techniques include using indexes, avoiding unnecessary joins, optimizing WHERE clauses, and ensuring proper indexing.
    • Example Query: N/A

  • Discuss the concept of SQL indexes.

    • Indexes are data structures that improve the speed of data retrieval operations on a database table.
    • Example Query: N/A

  • What is the purpose of the HAVING clause in SQL?

    • HAVING is used in conjunction with GROUP BY to filter grouped rows based on specified conditions.
    • Example Query: 
      SELECT department, AVG(salary) as avg_salary
        FROM employees
        GROUP BY department
        HAVING AVG(salary) > 50000;

  • Explain the difference between primary key and foreign key.

    • A primary key uniquely identifies a record in a table, while a foreign key links a column in one table to the primary key of another table, establishing a relationship.
    • Example Query: N/A

  • What is the role of the COMMIT statement in SQL?

    • COMMIT is used to save (or commit) changes made during a transaction, making them permanent.
    • Example Query: 
      BEGIN TRANSACTION;
        -- Perform updates, inserts, deletes
        COMMIT;

  • Discuss the purpose of the ORDER BY clause.

    • ORDER BY is used to sort the result set of a query based on specified columns and order (ascending or descending).
    • Example Query: 
      SELECT * FROM employees
        ORDER BY salary DESC;

  • How to perform a self-join in SQL?

    • A self-join is a regular join, but the table is joined with itself. It is often used when relating rows within the same table.
    • Example Query: 
      SELECT e1.name as employee, e2.name as manager
        FROM employees e1
        JOIN employees e2 ON e1.manager_id = e2.employee_id;

  • Explain the concept of a stored procedure.

    • A stored procedure is a set of SQL statements with a name, stored in the database, and executed as a single unit.
    • Example Query: 
      CREATE PROCEDURE GetEmployeeCount()
        AS
        BEGIN
            SELECT COUNT(*) FROM employees;
        END;

  • Discuss the use of triggers in SQL.

    • Triggers are sets of instructions that are automatically executed ("triggered") in response to specific events in the database.
    • Example Query: N/A

  • What is the purpose of the CASCADE keyword?

    • CASCADE is used in constraints (e.g., foreign key constraints) to specify that certain actions should propagate to related rows.
    • Example Query: N/A

  • How to use the LIMIT clause in SQL?

    • LIMIT is used to constrain the number of rows returned in a query result.
    • Example Query: 
      SELECT * FROM products
        LIMIT 10;

  • Explain the purpose of the BETWEEN operator.

    • BETWEEN is used to filter results within a specified range.
    • Example Query: 
      SELECT * FROM orders
        WHERE order_date BETWEEN '2022-01-01' AND '2022-01-31';

  • Discuss the significance of the UNION operator.

    • UNION is used to combine the result sets of two or more SELECT statements.
    • Example Query: 
      SELECT column1 FROM table1
        UNION
        SELECT column1 FROM table2;

  • What is the purpose of the LIKE operator?

    • LIKE is used in a WHERE clause to search for a specified pattern in a column.
    • Example Query: 
      SELECT * FROM customers
        WHERE name LIKE 'John%';

  • Explain the concept of SQL injection.

    • SQL injection is a code injection technique where malicious SQL statements are inserted into input fields, potentially leading to unauthorized access to the database.
    • Example Query: N/A

  • How to calculate the average in SQL?

    • AVG() is an aggregate function used to calculate the average value of a numeric column.
    • Example Query: 
      SELECT AVG(salary) FROM employees;

  • MySQL - Triggers:

    • Q: What is a trigger in MySQL?

    • A: A trigger is a set of instructions that are automatically executed (or "triggered") in response to specific events on a particular table.

    • Example:

      CREATE TRIGGER before_insert_example
            BEFORE INSERT ON employees
            FOR EACH ROW
            SET NEW.created_at = NOW();

  • MySQL - Stored Procedures:

    • Q: What is a stored procedure in MySQL?

    • A: A stored procedure is a set of SQL statements that can be stored in the database and executed later.

    • Example:

      DELIMITER //
            CREATE PROCEDURE get_employee_details (IN emp_id INT)
            BEGIN
                SELECT * FROM employees WHERE employee_id = emp_id;
            END //
            DELIMITER ;

  • MySQL - Views:

    • Q: What is a view in MySQL?

    • A: A view is a virtual table based on the result of a SELECT query, allowing users to query the view as if it were a table.

    • Example:

      CREATE VIEW active_employees AS
            SELECT * FROM employees WHERE status = 'active';

  • MySQL - Foreign Keys:

    • Q: What is a foreign key in MySQL?

    • A: A foreign key is a column or a set of columns that establishes a link between data in two tables, enforcing referential integrity.

    • Example:

      CREATE TABLE orders (
                order_id INT PRIMARY KEY,
                customer_id INT,
                FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
            );

  • MySQL - Indexing:

    • Q: Explain the purpose of indexing in MySQL.

    • A: Indexing enhances the speed of data retrieval operations on a database table by creating a data structure (index) that allows quick lookups.

    • Example:

      CREATE INDEX idx_last_name ON employees (last_name);

  • MySQL - Aggregate Functions:

    • Q: What are some commonly used aggregate functions in MySQL?

    • A: Aggregate functions include COUNT, SUM, AVG, MAX, and MIN.

    • Example:

      SELECT AVG(salary) FROM employees WHERE department_id = 1;

  • MySQL - Joins:

    • Q: Explain the difference between INNER JOIN and LEFT JOIN in MySQL.

    • A: INNER JOIN retrieves records where there is a match in both tables, while LEFT JOIN retrieves all records from the left table and matched records from the right table.

    • Example:

      SELECT * FROM employees
            INNER JOIN departments ON employees.department_id = departments.department_id;

  • MySQL - Subqueries:

    • Q: What is a subquery in MySQL?

    • A: A subquery is a query nested within another query, used to retrieve data that will be used by the main query.

    • Example:

      SELECT * FROM employees WHERE department_id IN (SELECT department_id FROM departments WHERE location = 'New York');

  • MySQL - UNION Operator:

    • Q: Explain the purpose of the UNION operator in MySQL.

    • A: The UNION operator combines the result of two or more SELECT statements into a single result set.

    • Example:

      SELECT employee_id, first_name FROM employees
            UNION
            SELECT customer_id, contact_name FROM customers;

  • MySQL - CASE Statement:

    • Q: What is the CASE statement in MySQL used for?

    • A: The CASE statement performs conditional logic in SQL queries.

    • Example:

      SELECT employee_id, salary,
                CASE
                    WHEN salary > 50000 THEN 'High'
                    WHEN salary > 30000 THEN 'Medium'
                    ELSE 'Low'
                END AS salary_range
            FROM employees;

    • MySQL - Data Types:

      • Q: What are some commonly used data types in MySQL?

      • A: Data types include INT, VARCHAR, DATE, FLOAT, and BOOLEAN.

      • Example:

        CREATE TABLE students (
                student_id INT,
                first_name VARCHAR(50),
                birth_date DATE,
                GPA FLOAT,
                is_enrolled BOOLEAN
            );

    • MySQL - GROUP BY Clause:

      • Q: Explain the purpose of the GROUP BY clause in MySQL.

      • A: The GROUP BY clause groups rows that have the same values in specified columns into summary rows.

      • Example:

        SELECT department_id, AVG(salary) FROM employees GROUP BY department_id;

    • MySQL - ROLLUP and CUBE:

      • Q: What is the ROLLUP and CUBE operators in MySQL?

      • A: ROLLUP and CUBE are used with the GROUP BY clause to generate subtotals and grand totals.

      • Example:

        SELECT department_id, job_id, AVG(salary)
            FROM employees
            GROUP BY ROLLUP (department_id, job_id);

    • MySQL - LIMIT Clause:

      • Q: How does the LIMIT clause work in MySQL?

      • A: The LIMIT clause is used to limit the number of rows returned by a query.

      • Example:

        SELECT * FROM employees LIMIT 10;

    • MySQL - Self-Join:

      • Q: How do you perform a self-join in MySQL?

      • A: A self-join is a regular join, but the table is joined with itself.

      • Example:

        SELECT e1.employee_id, e1.first_name, e2.supervisor_id
            FROM employees e1
            INNER JOIN employees e2 ON e1.employee_id = e2.supervisor_id;

    • MySQL - Auto Increment:

      • Q: What is the purpose of the AUTO_INCREMENT attribute in MySQL?

      • A: AUTO_INCREMENT is used to generate a unique numeric value automatically when a new record is inserted.

      • Example:

        CREATE TABLE students (
                student_id INT AUTO_INCREMENT PRIMARY KEY,
                first_name VARCHAR(50),
                last_name VARCHAR(50)
            );

    • MySQL - REPLACE Statement:

      • Q: Explain the REPLACE statement in MySQL.

      • A: The REPLACE statement is used to insert a new row or replace an existing row if a unique key violation occurs.

      • Example:

        REPLACE INTO employees (employee_id, first_name, last_name)
            VALUES (101, 'John', 'Doe');

    • MySQL - Full-Text Search:

      • Q: What is full-text search in MySQL?

      • A: Full-text search enables searching for text within columns of character data types.

      • Example:

        SELECT * FROM articles WHERE MATCH(title, content) AGAINST('MySQL full-text search');

    • MySQL - Foreign Key Constraints:

      • Q: How do foreign key constraints work in MySQL?

      • A: Foreign key constraints enforce relationships between tables, ensuring referential integrity.

      • Example:

        CREATE TABLE orders (
                order_id INT PRIMARY KEY,
                customer_id INT,
                FOREIGN KEY (customer_id) REFERENCES customers(customer_id) ON DELETE CASCADE
            );

    • MySQL - DATE and TIME Functions:

      • Q: What are some commonly used DATE and TIME functions in MySQL?

      • A: Functions include NOW(), DATE_FORMAT(), TIMESTAMPDIFF(), and CURDATE().

      • Example:

        SELECT * FROM events WHERE event_date > CURDATE();
