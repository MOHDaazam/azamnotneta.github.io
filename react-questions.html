Entry, Mid and Senior Level

  • How does React work?

    • React uses a virtual DOM to optimize rendering. It creates a virtual representation of the actual DOM and updates only the changed parts, reducing the need for direct manipulation of the real DOM.

  • What are the advantages of using React?

    • React provides a component-based architecture for reusability, a virtual DOM for efficient updates, one-way data binding for predictable state management, and a vibrant ecosystem.

  • What is the difference between a Presentational component and a Container component?

    • Presentational components are concerned with how things look, while Container components manage data and behavior. Container components may pass data and callbacks as props to presentational components.

  • What are the differences between a class component and functional component?

    • Class components use ES6 classes and have access to lifecycle methods, while functional components are simpler and introduced Hooks in React 16.8 for state and lifecycle management.

  • What is the difference between state and props?

    • State is internal to a component and can be changed, while props are external properties passed to a component. Props are immutable and passed down from parent to child.

  • What are the different lifecycle methods?

    • Common lifecycle methods include componentDidMount, componentDidUpdate, and componentWillUnmount. With Hooks, functional components use useEffect for similar functionality.

  • Explain React Hooks.

    • React Hooks are functions that enable the use of state and lifecycle features in functional components. Examples include useState, useEffect, and useContext. 
    import React, { useState, useEffect } from 'react';

function ExampleComponent() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]);

  return (
    <div>
      <p>You clicked {count} times.</p>
      <button onClick={() => setCount(count + 1)}>
        Click me
      </button>
    </div>
  );
}

  • What are controlled components?

    • Controlled components are components whose state is controlled by React. Their value is tied to the state, and any change triggers a re-render.
    import React, { useState } from 'react';

function ControlledComponent() {
  const [inputValue, setInputValue] = useState('');

  const handleInputChange = (event) => {
    setInputValue(event.target.value);
  };

  return (
    <input
      type="text"
      value={inputValue}
      onChange={handleInputChange}
    />
  );
}

    • Arrow functions in React provide a concise syntax and automatically bind the context, avoiding issues with this in class components.

  }

    return <button onClick={this.handleClick}>Click me</button>;
  }

  • What is Redux?

    • Redux is a state management library for JavaScript applications, particularly those built with React. It provides a predictable state container, actions, and reducers to manage and update application state.

  • What is a store in Redux?

    • The store in Redux is a single source of truth for the application state. It holds the complete state tree and allows components to subscribe and dispatch actions to modify the state.

  • Difference between action and reducer.

    • Actions are plain JavaScript objects describing changes to the state, while reducers are functions that specify how the state changes in response to actions. Actions are dispatched to trigger state modifications handled by reducers.
    // Action example

// Reducer example

  switch (action.type) {
    case 'INCREMENT':
      return state + 1;
    default:
      return state;
  }
};

  • Write a custom hook that can be used to debounce user's input.

    import { useState, useEffect } from 'react';

function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const handler = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => {
      clearTimeout(handler);
    };
  }, [value, delay]);

  return debouncedValue;
}

function useCopyToClipboard(text) {
  const [isCopied, setIsCopied] = useState(false);

  const copyToClipboard = () => {
    navigator.clipboard.writeText(text)
      .then(() => setIsCopied(true))

  • Write code for CRUD functionality in ReactJs?

    import React, { useState } from 'react';

function CrudExample() {
  const [items, setItems] = useState([
    { id: 1, name: 'Item 1' },
    { id: 2, name: 'Item 2' },
    // ... other items
  ]);

  const handleAdd = () => {
    const newItem = { id: items.length + 1, name: `Item ${items.length + 1}` };
    setItems([...items, newItem]);
  };

  const handleUpdate = (id, newName) => {
    setItems(items.map(item => (item.id === id ? { ...item, name: newName } : item)));
  };

  const handleDelete = (id) => {
    setItems(items.filter(item => item.id !== id));
  };

  return (
    <div>
      <ul>
        {items.map(item => (
          <li key={item.id}>
            {item.name}
            <button onClick={() => handleUpdate(item.id, `Updated ${item.name}`)}>Update</button>
            <button onClick={() => handleDelete(item.id)}>Delete</button>
          </li>
        ))}
      </ul>
      <button onClick={handleAdd}>Add Item</button>
    </div>
  );
}

  • What is a hook in React and why are they useful?

    • Hooks in React are functions that enable the use of state and lifecycle features in functional components. They provide a cleaner and more concise way to manage component logic compared to class components.

  • What are some common hooks that are used in React?

    • Commonly used hooks in React include useState, useEffect, useContext, useReducer, useCallback, useMemo, useRef, and useEffect.

  • Can you use hooks inside a class-based component?

    • No, hooks are designed to work only in functional components and not in class-based components. Using hooks in a class component would result in an error.

  • How do you test a component that uses hooks?

    • Components using hooks can be tested using testing libraries like Jest and React Testing Library. Mocking the hook functions and testing component behavior, state changes, and UI interactions are common approaches.

    import { useState } from 'react';

function useCounter(initialValue, step) {
  const [count, setCount] = useState(initialValue);

  const increment = () => setCount(count + step);
  const decrement = () => setCount(count - step);

  return { count, increment, decrement };
}

// Usage

  const { count, increment, decrement } = useCounter(0, 1);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>Increment</button>
      <button onClick={decrement}>Decrement</button>
    </div>
  );
}

  • What is the difference between useEffect and useLayoutEffect?

    • useEffect runs after the browser has painted, while useLayoutEffect runs synchronously after all DOM mutations. Use useEffect for less critical side effects, and useLayoutEffect when you need to perform actions that require layout calculations before painting.

  • Why is the virtual DOM faster to update than the real DOM?

    • The virtual DOM is faster because it creates a lightweight, in-memory representation of the actual DOM. React can efficiently update the virtual DOM, identify the minimal changes needed, and apply those changes to the real DOM in a batch, minimizing reflows and repaints.

  • Can you explain the difference between a pure and impure function, and why it matters in the context of React?

    • A pure function produces the same output for the same input and has no side effects, making it predictable and easy to test. In React, pure functions are preferred for components to ensure predictable rendering and behavior.
    // Pure function
function add(a, b) {
  return a + b;
}

// Impure function
let result = 0;
function impureAdd(a, b) {
  result = a + b;
  return result;
}

  • Styled-Components vs Inline Styling in React?

    • Styled-Components is a popular CSS-in-JS library that allows writing styled components with tagged template literals. It provides more control, better organization, and supports dynamic styling. Inline styling, on the other hand, involves directly adding styles to JSX elements using the style attribute. While convenient, it can become difficult to manage in large applications and lacks the features provided by Styled-Components. Using Styled-Components is generally recommended for scalable styling in React.
